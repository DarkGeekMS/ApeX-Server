filterResult()

filterResult(\Illuminate\Support\Collection  $result, \App\Http\Controllers\JWT  $token, boolean  $subscribed = false) : \Illuminate\Http\Response

Just a helper fuction to remove posts from blocked users, posts that are hidden or reported by the current user and posts from apexComs that the current user is blocked from and remove blocked users and apexComs from the result it also adds the current user votes on the posts and if he had saved the post previously, if the bool `subscribed` is true, limit the posts to be from only apexComs that the user is subscribed in it also adds the current user subscription of the apexComs

guestSortPostsBy()

guestSortPostsBy(\Illuminate\Http\Request  $request) : \Illuminate\Http\Response|array

Get the posts sorted by date, votes or comments.

Validate the input by checking that the given apexComID exists, and the sortingParam is one of [votes, date, comments], if it's not, it uses date as a default value. If the apexComID is not found it return an error, else it uses that id to get the posts in that apexCom if there is no subscribedApexCom parameter in the request (not called from userSortPosts), then it return the posts sorted by the given sortingParam.
